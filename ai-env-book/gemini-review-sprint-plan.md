# Gemini Review Sprint Plan: ai-env-book

> **Tactical execution for the Reviewer Team.**
> This plan follows the **AgentFlow methodology for writing**: Research -> Strategy -> Execution (Plan -> Act -> Validate).
> The goal is to produce a comprehensive `gemini_says.md` artifact to sharpen the `ai-env-book` manuscript.

---

## Sprint 0 — Environment & Setup (Research Phase)

**Goal**: Prepare the review environment and ensure all agents and skills are aligned with the current manuscript.

| Task | Indicator of Done |
| :--- | :--- |
| **Research**: Audit all 17 chapter files and the combined manuscript in `ai-env-book/chapters/`. | A verified "Chapter Map" exists in the session context. |
| **Strategy**: Assign specific "Attack Vectors" for each chapter to the 5-agent team. | A "Review Matrix" is created, mapping agents to specific chapters and focal points. |
| **Execution (Plan)**: Define the structure of the `gemini_says.md` template. | `gemini_says.md` exists with placeholders for all 5 agent sections. |
| **Execution (Act)**: Initialize `gemini_says.md` with current high-level project goals. | `gemini_says.md` is committed with the project's core thesis and audience. |
| **Execution (Validate)**: Verify all 5 review agents and skills are correctly loaded and accessible. | A "Hello World" verification from each agent is confirmed in the session. |

## Exit Criteria

- `gemini_says.md` exists and is comprehensive across all 15 chapters + intro/conclusion.
- Every note in `gemini_says.md` is **actionable** (not just descriptive).
- All 5 reviewer agent personas are represented in the final report.
- The "Technical Red Zone" (landmines) and "Green Zone" (verified) are clearly delineated.
- The "AI Edge" section provides clear instructions for the Writer AI to improve its output.
