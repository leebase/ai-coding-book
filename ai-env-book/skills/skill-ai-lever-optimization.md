# Skill: AI Lever Optimization

## Goal
Maximize the value of environment knowledge for AI-assisted development.

## Directives

### 1. Grounding Heuristics
- Does the chapter show the reader how to "ground" an AI by providing it with environment-specific files (e.g., `.gitignore`, `pyproject.toml`)?
- Identify sections where the reader can copy-paste environment output into an AI for analysis.

### 2. The AI-Connection Callouts
- Every chapter should have at least one "AI Tip" callout explaining how the current environment concept improves AI performance.
- Example: "Knowing your Git branch structure allows you to tell the AI exactly which context to ignore."

### 3. Error Recovery Patterns
- Teach the reader to treat terminal errors as "AI Prompts."
- Show how to feed a `ModuleNotFoundError` or a Git merge conflict into an AI to generate a fix.

### 4. Contextual Awareness
- Help the reader understand that the AI *is* an agent in their environment.
- Frame the environment as the "API" through which the AI interacts with the code.
