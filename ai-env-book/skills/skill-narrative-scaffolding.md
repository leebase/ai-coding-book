# Skill: Narrative Scaffolding

## Goal
Ensure a coherent, reinforced learning journey using the scenario-driven project thread.

## Directives

### 1. Scenario Continuity
- Verify that the reader's project state (e.g., the `task-manager` directory) matches across chapters.
- Ensure that a file created in a "Part 1" chapter is still relevant or correctly referenced in "Part 3."

### 2. Reinforcement Loops
- Identify "Recall points" where the reader is asked to use a skill from a previous chapter to solve a new problem.
- Ensure the difficulty increases incrementally—never introduce two high-friction concepts in one chapter.

### 3. Voice Consistency
- Audit for "Tone Drift." The book should sound like a trusted mentor, not a dry manual or a marketing brochure.
- Ensure the reader persona (a developer building their first AI-assisted project) is consistently addressed.

### 4. The "Scenario Thread" Guard
- If a chapter deviates from the central scenario to explain a concept, ensure it "hooks back" into the project within two pages.
