# Skill: Framework Design

Use this skill when creating named models, diagnostic tools, or visual
structures for a nonfiction book or consulting method.

## A Good Framework Must

- Clarify the argument, not decorate it
- Be simple enough to remember
- Create useful distinctions
- Suggest action, not just classification
- Hold up under skeptical questioning

## Design Checklist

- What problem does this framework solve for the reader?
- What decisions does it help them make?
- What are the parts, and why these parts?
- What would a smart critic say is missing?
- Does the name earn its place?

## Failure Modes

- Fancy labels for obvious ideas
- Too many buckets to remember
- A model that sounds smart but does not change action
- Naming the framework before the logic is clear
