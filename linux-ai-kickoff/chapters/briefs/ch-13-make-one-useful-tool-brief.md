# Chapter 13 Brief - Make One Useful Tool

## Mission

- Mission: Help the reader turn the launchpad into leverage by creating one tiny
  helper script they actually use.
- Why it matters: Chapter 12 gave the reader a place and a purpose. This
  chapter should make returning to that place easier, faster, and more
  automatic.
- Success looks like: The reader creates and runs one small script that takes
  them back to their launchpad goal folder and shows them their mission note.
- Next door it opens: Chapter 14 can troubleshoot a real break or friction
  point inside a workspace the reader is now actively reusing.

## Purpose

This chapter needs to:

1. make the idea of a "tool" feel small and doable
2. keep using the same launchpad and `mission.md` from Chapter 12
3. teach a tiny script as leverage, not as programming identity
4. keep the commands safe and followable
5. end with a helper the reader can use again tomorrow

## Working Example Set

Primary helper:

- a small `start-goal.sh` script in `~/launchpad`
- it is sourced into the current shell so it changes into the goal folder
- it prints the mission note

Core moves:

1. read the relevant manual sections first
2. ask AI to help shape the tiny script
3. create the file
4. run it and observe what happens

Keep out of chapter:

- shell startup-file editing
- long Bash explanations
- command-line wizardry
- multiple tools in one chapter
- turning the script into a full app

## Working Outline

1. Why one tiny tool matters
2. Read the manual and choose script over alias for now
3. Ask AI for a tiny helper shaped to the launchpad
4. Create and run `start-goal.sh`
5. End with a helper the reader can keep using
