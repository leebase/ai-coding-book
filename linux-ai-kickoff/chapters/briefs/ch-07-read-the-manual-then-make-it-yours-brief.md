# Chapter 7 Brief - Read the Manual, Then Make It Yours

## Mission

- Mission: Teach the reader the manual-first loop directly by helping them use
  the Omarchy manual, AI, and one safe terminal experiment to solve a real
  small problem.
- Why it matters: If the reader only asks AI from zero, they never build the
  habit of checking the source of truth or recognizing what good instructions
  look like.
- Success looks like: The reader reads the right manual section, asks AI to
  explain it in plain English, tries one safe command or workflow, and learns
  what to do next from what happened.
- Next door it opens: Part 3 can teach AI more directly because the reader has
  already seen AI work best as a tutor grounded by docs.

## Purpose

This chapter needs to:

1. make RTFM feel like power, not homework
2. teach a repeatable loop, not just one command
3. keep AI in the role of explainer and helper
4. use a safe, real Omarchy-manual example
5. show how the same loop scales to bigger system and development tasks later

## Source Notes

Primary manual sections:

- `Navigation`
- `AI`
- `Development Tools`
- `Shell Tools`

Primary example flow:

- use `Navigation` to put browser and terminal side by side
- read the `Shell Tools` section for one narrow problem: finding files or text
- ask AI to explain the difference between `ff`, `fd`, and `rg` in plain
  English for a beginner
- try one safe experiment with `ff`
- reflect on what happened and what tool to use next time

Supporting points:

- `AI` section reinforces that Omarchy ships with harnesses, but the manual
  still supplies the grounding and cautions
- `Development Tools` shows that this same manual-first loop scales to bigger
  choices like editors, environments, and GitHub CLI later

Keep out of chapter:

- full harness setup or advanced prompting
- danger mode or live agent edits
- deep comparisons of all shell tools
- editor wars or environment-install rabbit holes
- config-file customization beyond the chapter's narrow example

## Working Outline

1. Manuals are how you stop guessing
2. Pick one narrow question worth answering
3. Open browser and terminal side by side
4. Read the relevant Omarchy-manual section first
5. Ask AI to translate and compare, not replace the source
6. Try one safe shell-tools experiment
7. End by naming the reusable loop and how it scales
