# Chapter 4 Brief - Meet the Machine

## Mission

- Mission: Make Omarchy feel usable on day one by teaching the reader the few
  entry points that matter most.
- Why it matters: If the system feels like a wall, the reader will bounce off
  the machine before the later chapters can build confidence.
- Success looks like: The reader can open the launcher, the Omarchy menu, the
  hotkey sheet, and the core daily apps they need to begin.
- Next door it opens: Chapter 5 can focus on moving through the system with
  growing fluency.

## Purpose

This chapter needs to:

1. lower intimidation immediately
2. teach only the smallest useful set of system entry points
3. frame the keyboard-first model as leverage, not weirdness
4. point the reader back to the official manual as a living map

## Source Notes

Primary manual sections:

- `Welcome to Omarchy`
- `Navigation`
- `Hotkeys`

Key actions to ground:

- application launcher: `Super + Space`
- Omarchy menu: `Super + Alt + Space`
- hotkey sheet: `Super + K`
- terminal: `Super + Return`
- browser: `Super + Shift + Return`

## Working Outline

1. The system may feel strange because it is keyboard-first
2. You do not need the whole map, just the main doors
3. Learn the launcher, menu, and hotkey sheet
4. Open the terminal and browser once
5. End with a small orientation drill
