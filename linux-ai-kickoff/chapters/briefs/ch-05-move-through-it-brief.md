# Chapter 5 Brief - Move Through It

## Mission

- Mission: Teach the reader the minimum movement patterns that make Omarchy
  feel navigable instead of intimidating.
- Why it matters: If the reader can enter the system but not move through it,
  the machine still feels like something fragile and foreign.
- Success looks like: The reader can switch workspaces, close and refocus
  windows, and use the unified clipboard with confidence.
- Next door it opens: Chapter 6 can introduce the terminal from a stronger base
  of orientation.

## Purpose

This chapter needs to:

1. teach movement, not encyclopedic hotkey recall
2. focus on a small set of reusable navigation patterns
3. make the clipboard feel like relief, not trivia
4. keep the manual visible as the place to check the wider map

## Source Notes

Primary manual sections:

- `Navigation`
- `Unified clipboard`
- `Hotkeys`

Key actions to ground:

- next workspace: `Super + Tab`
- previous workspace: `Super + Shift + Tab`
- jump to specific workspace: `Super + 1/2/3/4`
- close window: `Super + W`
- toggle fullscreen: `Super + F`
- copy/paste: `Super + C`, `Super + V`
- clipboard history: `Super + Ctrl + V`

## Working Outline

1. Movement reduces hesitation
2. Workspaces are rooms, not a mystery
3. Learn the close and fullscreen controls
4. Use the unified clipboard and clipboard history
5. End with a short navigation drill
