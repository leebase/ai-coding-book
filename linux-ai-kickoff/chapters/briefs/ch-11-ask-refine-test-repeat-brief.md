# Chapter 11 Brief - Ask, Refine, Test, Repeat

## Mission

- Mission: Teach a simple practical loop for learning with AI so the reader
  stops hoping for one perfect answer and starts building progress through
  asking, refining, testing, and repeating.
- Why it matters: Chapters 8 through 10 reframed AI, taught the Team-Captain
  model, and normalized drift. This chapter now turns those ideas into a
  repeatable working method.
- Success looks like: The reader completes one full loop on a small real task
  and can point to the ask, the refine step, the test, and the repeat.
- Next door it opens: Part 4 can use this loop inside real missions and real
  practical output.

## Purpose

This chapter needs to:

1. name the four-step loop in plain English
2. show where reality enters the process
3. keep the example safe, small, and real
4. distinguish testing from blind trust
5. end with one loop the reader can run immediately

## Working Example Set

Primary example:

- create a `practice` folder in the home directory
- confirm the terminal is inside it
- ask one follow-up question based on what happened

Core loop:

1. Ask
2. Refine
3. Test
4. Repeat

Keep out of chapter:

- deep debugging
- complex scripting
- prompt jargon
- long shell detours
- stealing Part 4's full project thread

## Working Outline

1. Why one-shot hope is weak
2. The four-step loop
3. One worked example from first ask to repeat
4. What to do when the test surprises you
