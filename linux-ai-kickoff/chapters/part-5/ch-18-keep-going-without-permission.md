# Keep Going Without Permission

By now, you have something a lot of people never build.

You have proof.

You have a direction.

You have a small curriculum.

That is already more than motivation.

Now the question is simpler:

how do you keep going when nobody is assigning the work?

That is what "without permission" means here.

Not rebellion.

Not drama.

Not pretending you know exactly where your life is headed.

It means you stop waiting for:

- a class
- a boss
- a credential
- a perfect plan
- the feeling that you are finally ready

before you do the next real piece of work.

By the end of this chapter, you should have a `next-three-moves.md` file in the
same launchpad folder, with three concrete moves in order and Move 1 either
started now or scheduled.

## Do Not Rebuild The Whole Future

This chapter is not asking you to choose a new direction.

Chapter 16 already did that.

It is not asking you to build a new curriculum.

Chapter 17 already did that too.

This chapter has one job:

shrink the future into the next three moves.

That is enough.

Most people get stuck because they try to solve too much at once.

They think:

what is the right field?

what if I change my mind?

what if this does not lead anywhere?

Those are real questions.

But they are usually not the next questions.

The next question is:

what are the next three moves that keep me in motion?

## The Permissionless Rule

When nobody is assigning the work, use this rule:

1. do the next real move
2. keep one short note about what happened
3. choose the next move from evidence

That is enough to keep learning.

You do not need more approval.

You do not need a formal program.

You need a way to continue.

## Start From `curriculum.md`

Open the `curriculum.md` file from Chapter 17.

Look at the three missions you picked.

Now do something smaller and more useful:

turn those missions into the next three moves only.

A move is not a life decision.

A move is one concrete thing you can do in the next few days.

It should be small enough to survive real life.

It should be clear enough that you know when it is done.

It should connect to the direction and curriculum you already chose.

## Give Each Move A Job

Each move should answer four things:

- what you are doing
- when you will do it
- what supports it
- what done looks like

That is what keeps the move real.

For support, use the same habit this book has been teaching the whole time:

- a primary doc or source
- the tool you will use
- AI if you need clarification, comparison, or help shrinking the next step

AI can be part of support here, but it is not required.

Do not write moves like:

`work on automation`

`get better at writing`

`learn more troubleshooting`

Those are not moves.

These are moves:

`Write the notes-file helper tomorrow at 7 PM. Doc: Omarchy shell tools section. Tool: shell. Done looks like: one helper file that works twice in a row.`

`Draft one 300-word explanation on Saturday morning. Doc: Markdown reference. Tool: my note system. Done looks like: one note I would let another person read.`

That is the level you want.

## Make The Three Moves Different

If all three moves are the same size, the plan can feel flat.

It helps to give them different jobs.

One good pattern is:

- Move 1: do one real thing now
- Move 2: keep one short note about what happened or what you learned
- Move 3: choose and set up the next mission based on what Move 1 taught you

That pattern works because it keeps action, learning, and adjustment together.

You are not just pushing forward blindly.

You are paying attention.

## What To Do When You Stall

You will stall sometimes.

That is normal.

When that happens, do not turn it into a referendum on your future.

Use a smaller loop.

1. shrink the move
2. check the primary doc
3. ask AI one specific question
4. do one 20-minute attempt
5. write one sentence about what happened

That is enough to get unstuck surprisingly often.

You do not need a heroic recovery.

You need a smaller next move.

## Write `next-three-moves.md`

Now create one more file in the same launchpad goal folder:

`next-three-moves.md`

Use this shape:

```md
# Next Three Moves

## Direction
[paste the sentence from direction.md]

## Move 1
[the first concrete move]
- When: [today / tomorrow / exact time]
- Doc: [doc or source]
- Tool: [tool]
- Done looks like: [visible result]

## Move 2
[the next concrete move]
- When: [specific window]
- Doc: [doc or source]
- Tool: [tool]
- Done looks like: [visible result]

## Move 3
[the next concrete move]
- When: [specific window]
- Doc: [doc or source]
- Tool: [tool]
- Done looks like: [visible result]

## Start Now
[the one tiny thing you will do in 2-5 minutes before you close the file]
```

That is enough.

It does not solve the next five years.

It gives you a live edge to work from.

## One Worked Example

Say your `curriculum.md` is about automation.

Your `next-three-moves.md` might look like this:

```md
# Next Three Moves

## Direction
A direction worth exploring next is automation, because building one tiny
helper made my machine feel useful instead of passive.

## Move 1
Write the notes-file helper.
- When: tomorrow at 7 PM
- Doc: Omarchy manual section on shell tools
- Tool: shell
- Done looks like: one helper file that creates the note twice in a row

## Move 2
Write one note about what was confusing and what worked.
- When: right after Move 1
- Doc: the command docs I actually used
- Tool: note system
- Done looks like: one saved note I can reuse later

## Move 3
Choose the next helper based on what I kept doing by hand.
- When: this weekend
- Doc: curriculum.md
- Tool: AI chat for narrowing
- Done looks like: one clearly named next helper mission

## Start Now
Make the helper file and write the first line.
```

Notice what this example does not do.

It does not wait for confidence.

It just keeps the path alive.

## A Non-Tech Example

This works outside tech too.

Say your direction is learning local history well enough to publish a small
guide for your town.

Your three moves could look like this:

- Move 1: collect and summarize three local sources
  - When: Saturday morning
  - Doc: the library archive guide
  - Tool: note system
  - Done looks like: one note with three source summaries
- Move 2: draft one short profile of a place or person
  - When: Sunday afternoon
  - Doc: the source note
  - Tool: AI chat for outline help
  - Done looks like: one short profile you would show somebody
- Move 3: choose whether the next small project is another profile or one page
  guide
  - When: after Move 2
  - Doc: your curriculum note
  - Tool: note system
  - Done looks like: one named next mission

Same pattern.

Different direction.

That matters.
