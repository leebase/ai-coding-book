# Sprint Plan: Teach Yourself Anything

> Tactical execution plan for the full book. For the strategic roadmap, see
> `project-plan.md`.

---

## Production Assumptions

This sprint plan covers the whole production of the book from architecture to
publication.

To make the plan concrete, it uses the following **working manuscript shape**:

- Introduction
- Part 1: 3 chapters
- Part 2: 4 chapters
- Part 3: 4 chapters
- Part 4: 4 chapters
- Part 5: 3 chapters
- Conclusion

That is **19 manuscript files total**.

The exact chapter titles may change in Sprint 1, but the production runway
assumes a book of this size so planning, review, and publication work are fully
scoped now.

---

## Standard Chapter Definition of Done

A chapter sprint is complete only when all of the following are true:

1. Chapter purpose and mission are explicit
2. Visible reader outcome is named
3. Manual-first opportunities are identified where relevant
4. Draft reflects the book's agency-centered thesis
5. `pathfinder` review is addressed
6. `confidence-auditor` review is addressed
7. `manual-guide` review is addressed for technical chapters
8. `mission-designer` review is addressed
9. Beginner-friction review is addressed where relevant
10. Chapter is added to the current manuscript structure without continuity drift

---

## Standard Production Sequence Per Chapter

Every drafting sprint follows the same order:

1. Define the chapter mission and reader outcome
2. Map any required primary sources
3. Create chapter brief and outline
4. Draft chapter
5. Run custom-agent review
6. Revise chapter
7. Run continuity check against adjacent chapters
8. Mark sprint complete in `context.md` and `result-review.md`
