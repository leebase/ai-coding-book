# Skill: Manual First

Use this skill when a chapter, outline, or example depends on technical truth
that should come from an official source rather than memory.

For this book, the default primary source is:

- [The Omarchy Manual](https://learn.omacom.io/2/the-omarchy-manual)

## Core Rule

Read the manual first. Use AI second.

AI can explain, simplify, compare, and help apply. AI should not replace the
act of checking the source.

## Required Sequence

1. Identify the exact manual section or primary source
2. Extract only the part relevant to the current chapter mission
3. Say why that section matters to the reader right now
4. Use AI to clarify or apply what the source says
5. Point the reader back to the source so they learn the habit

## What Good Looks Like

- The chapter names the source section, not just "the docs"
- The source feels like a tool, not homework
- The reader sees how manuals reduce confusion
- AI is shown helping with understanding and next actions

## What To Avoid

- paraphrasing from memory when the manual should be checked
- dumping long documentation into the chapter
- treating manuals as punishment
- letting AI summarize a source the chapter never names
