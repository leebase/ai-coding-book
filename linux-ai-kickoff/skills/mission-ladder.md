# Skill: Mission Ladder

Use this skill when designing a chapter mission or practical exercise.

This book works by stacking small wins into confidence. A chapter mission
should feel meaningful, finishable, and connected to a bigger path.

## Mission Checklist

Every mission should answer:

1. What is the reader doing?
2. Why does it matter in real life?
3. What visible outcome proves they did it?
4. What could go wrong?
5. What new door does this open?

## Difficulty Rules

- Start smaller than your expert instinct wants
- Prefer one visible win over three partial wins
- Never ask for a later-rung skill before the earlier rung exists
- Escalate by confidence, not by impressiveness

## Good Mission Shapes

- create one useful file or workspace
- solve one real friction point
- customize one thing that makes daily use better
- use AI to understand and fix one confusing problem
- build one tool the reader actually keeps

## Weak Mission Shapes

- "understand the system better"
- "learn Linux basics"
- "become familiar with AI"
- anything with no visible outcome

## Output Format

When using this skill, write the mission in four lines:

- Mission:
- Why it matters:
- Success looks like:
- Next door it opens:
